## Abstract
Deploying image processing workloads on NVIDIA GPUs at near speed-of-light performance has required hand-tuning kernels using CUDA C++, given the unique nature of the loads used in semiconductor manufacturing. However, maintaining and debugging CUDA C++ code is challenging, often requiring significant re-tuning when adopting new GPU architectures. Tile-based programming, enabled by domain-specific languages (DSLs) such as NVIDIA cuTile and OpenAI Triton, offers a more portable and maintainable alternative while achieving performance close to hand-tuned CUDA C++.

We'll evaluate tile-based programming for image processing workloads used in semiconductor manufacturing, comparing development effort and performance when porting operations from CUDA C++ to cuTile and Triton. We share insights from implementing GPU-accelerated algorithms, debugging, and optimizing performance in these DSLs.
